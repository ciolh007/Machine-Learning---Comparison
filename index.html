Skewed Classes

At the moment, we have only 11 examples of class 'red'

The data is highly skewed

If my classifier returns class='blue' for all instances, then it will be mostly accurate already

Overcoming this problem

  1. Get more or better data
  2. If that is not possible, data augmentation methods (modifying existing instances by varying some of the attributes) may be employed
  3. It may be sufficient to copy the instances to balance the number of instances per class

Cross Validation

So far, the accuracy scores are only meaningful for debugging, that is making sure that your code does not crash

Next, we want to make sure we can evaluate the performance correctly, using either hold-out or k-fold cross validation

[0.72 0.64 0.72 0.56]

N_DIM = 3

Xs_ext = np.zeros( ( len(Xs), 1 + 2 * N_DIM ) )
Xs_ext[:,0] = 1.0

for i in range(N_DIM):
  Xs_ext[:,1 + 2*i] = Xs[:,0] ** i
  Xs_ext[:,1 + 2*i + 1] = Xs[:,1] ** i

Leaking Training Data

After shuffling the training data, the performance estimate is much more realistic

In order to estimate the generlization performance over multiple runs, we often necessary to split the data into three sets, not just two sets: training data, testing data, validation data

All sets should be uncorrelated

Validation data allows us to answer questions like: How often will you be able to learn a 90% accurate classifier, given this domain

Conclusion

The generalization ability of the various algorithms is pretty similar (tested using cross-validation)

In the late 90's, a lot of work was done to try and find the best machine learning algorithm. For some domains, there may be algorithms that perform very well, but over a large range of domains the performance is often very similar

The most important change in our small study was the realization that the training and test datasets were correlated using cross-validation

Nowadays, we realize that the most important aspect of a ML system is the training data, both quantity and quality of the data
